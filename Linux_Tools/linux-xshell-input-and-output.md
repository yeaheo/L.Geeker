## 关于Shell中的输出\输入问题
- 默认情况下，总是有三个文件处于打开状态，标准输入(键盘输入)、标准输出（输出到屏幕）、标准错误（也是输出到屏幕），它们分别对应的文件描述符是 0，1，2 
- 下面是几种常见情况：
  - `>/dev/null 2>&1`
    -  实际上，应该等同于这样： 1>/dev/null 2>/dev/null
    -  即：标准输出重定向到了/dev/null，而标准错误又重定向到了标准输出，所以就成了标准输出和标准错误都重定向到了/dev/null
  - `2>&1 >/dev/null` 
    - 标准错误打印到屏幕，而标准输出不打印到屏幕  
    
  - `&>/dev/null`
    - 结果通通重定向到/dev/null
